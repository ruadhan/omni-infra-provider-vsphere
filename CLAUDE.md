## What this is

An Omni infrastructure provider for vSphere: it connects to an Omni instance as an infra provider and provisions Talos (>= v1.12) nodes by cloning VMs from an OVA template (built via factory.talos.dev) in vCenter. Single Go binary at `cmd/omni-infra-provider-vsphere`.

## Build system: kres-generated — do not hand-edit

The `Makefile`, `Dockerfile`, `.golangci.yml`, `.conform.yaml`, and `.github/workflows/ci.yaml` are generated by [kres](https://github.com/siderolabs/kres) and marked "DO NOT EDIT". To change them, edit `.kres.yaml` and run `make rekres`. All make targets run inside Docker buildx (a builder instance must exist: `docker buildx create --name local --use`).

## Architecture

- `cmd/omni-infra-provider-vsphere/main.go` — cobra entrypoint. Reads YAML config (`internal/pkg/config`, vSphere URI/user/password), creates a govmomi client with a SOAP keep-alive handler, then registers the provisioner with Omni via `siderolabs/omni/client/pkg/infra` (`infra.NewProvider` + `ip.Run`). Embeds `data/schema.json` (machine-class form schema shown in the Omni UI) and `data/icon.svg`.
- `internal/pkg/provider/provision.go` — the core. `Provisioner` implements the Omni infra provisioner interface:
  - `ProvisionSteps()` returns two idempotent steps: **createVM** (resolve datacenter/folder/pool/template/datastore via `find.Finder`, clone from the template, resize first disk, swap the NIC network, optionally apply an SPBM storage policy) and **powerOnVM**.
  - `Deprovision()` powers off and destroys the VM; it relies on the datacenter stored in machine state and treats "VM not found" as success.
  - Talos join config, hostname patch, and optional CA-cert patch are concatenated into multi-document YAML, base64-encoded, and injected via the `guestinfo.talos.config` extraConfig key — the template must not have Omni join kernel args baked in.
  - Retryable failures return `provision.NewRetryErrorf(...)` (Omni re-invokes the step); every step and Deprovision first calls `p.ensureSession()` to re-login if the vSphere session expired.
  - Storage policy note: the home-level relocate `Profile` does not cover VMDKs, so per-disk `VirtualMachineRelocateSpecDiskLocator` entries are built too (`diskProfileLocators`).
- `internal/pkg/provider/data.go` — `Data` is the machine-class `providerdata` YAML. **When adding a field, keep three places in sync**: this struct, `cmd/omni-infra-provider-vsphere/data/schema.json`, and the field table in `README.md`.
- `api/specs/specs.proto` — `MachineSpec`, the per-machine state persisted in Omni (VM name, datacenter). Regenerate `*.pb.go` with `make generate`. Wrapped as a COSI typed resource in `internal/pkg/provider/resources/machine.go`.
- `test/` — example machine class and cluster template YAML for manual use with `omnictl`, not automated tests.
